Paleta de Colores Corporativa
Variables CSS personalizadas — Custom Properties
Explicación

La paleta de colores se define mediante variables CSS (Custom Properties) en el selector :root. Esto permite reutilizar los colores corporativos en todo el sitio desde un único punto de control.

Fucsia (#97336F): Color principal de marca. Se usa en títulos H1/H2, categorías, links activos y acentos visuales.

Burdeos (#471834): Fondo secundario y componente oscuro del gradiente corporativo.

Naranja (#E0994E): Color de acción. Se aplica en botones CTA, precios y elementos que requieren atención inmediata.

Negro (#000000): Fondo principal de todas las páginas, aporta contraste y elegancia.

CSS Custom Properties :root selector Identidad Visual

/* Variables de color corporativo */
:root {

    --burdeos:  #471834;   /* Fondos secundarios */
    --naranja:  #E0994E;   /* Botones y CTAs */
    --negro:    #000000;   /* Fondos */
    --blanco:   #FFFFFF;   /* Texto sobre oscuros */

    /* Gradiente corporativo */
    --gradiente: linear-gradient(
        160deg,
        #471834 0%,
        #97336F 100%
    );
}

Explicación

Montserrat se usa para todos los títulos (H1–H3), etiquetas y elementos de navegación. Su estilo geométrico y bold transmite profesionalismo y modernidad.

Iconografía SVG
Gráficos vectoriales escalables — Accesibilidad con aria-label
Explicación

Todos los iconos del sitio están codificados en formato SVG inline (Scalable Vector Graphics). A diferencia de imágenes PNG o JPG, los SVG:

— Se escalan sin perder calidad a cualquier resolución
— Pueden ser coloreados con CSS usando fill y stroke
— Son accesibles con aria-label para lectores de pantalla
— No requieren peticiones HTTP adicionales al ser inline

Se implementaron iconos para: búsqueda, usuario, corazón, carrito, redes sociales y secciones de beneficios.

SVG inline fill / stroke aria-label Scalable

Botones e Interacción — Efecto Hover y Ripple
Retroalimentación visual inmediata al usuario
Explicación

El sitio tiene tres variantes de botón según su jerarquía de acción:

Primario (naranja): Para acciones principales como "Comprar Ahora" o "Suscribirme". Usa background: var(--naranja) y borde redondeado.

Secundario (borde): Para acciones alternativas como "Asesoría". Usa borde visible con fondo transparente.

Fucsia: Para acciones de carrito y elementos de marca.

Todos los botones tienen efecto ripple codificado en JavaScript: al hacer clic se crea dinámicamente un elemento <span> con animación de onda que se elimina automáticamente.

transition transform Efecto Ripple createElement
styles.css + main.js 
/* CSS — Botón primario */
.btn-primary {
    background-color: var(--naranja);
    color: var(--blanco);
    padding: 14px 36px;
    border-radius: 30px;
    transition: all 0.3s;
    position: relative;
    overflow: hidden; /* Necesario para ripple */
}
.btn-primary:hover {
    transform: translateY(-3px);
    box-shadow: 0 8px 20px rgba(0,0,0,0.2);
}

/* JS — Algoritmo efecto ripple */
btn.addEventListener('click', function(e) {
    const ripple = document.createElement('span');
    ripple.className = 'ripple-effect';
    const rect = this.getBoundingClientRect();
    const size = Math.max(rect.width, rect.height);
    ripple.style.width = ripple.style.height = size + 'px';
    ripple.style.left = (e.clientX - rect.left - size/2) + 'px';
    ripple.style.top  = (e.clientY - rect.top  - size/2) + 'px';
    this.appendChild(ripple);
    setTimeout(() => ripple.remove(), 600);
});

05
Menú Hamburguesa Responsive
Navegación móvil con overlay — CSS + JavaScript
Explicación del algoritmo

El menú hamburguesa se activa en dispositivos con ancho ≤ 768px mediante una media query CSS. El botón está oculto en desktop (display: none) y visible en móvil (display: flex).

Algoritmo JavaScript:
1. Al cargar la página, el JS verifica si el botón existe en el DOM
2. Si no existe, lo crea dinámicamente con createElement
3. Al hacer clic, alterna las clases .nav-open (en el nav) y .active (en el botón)
4. Se crea un overlay oscuro semitransparente detrás del menú
5. El menú se cierra con: clic en overlay, clic en enlace, o tecla Escape

Las 3 líneas del botón se transforman en una X animada con rotaciones CSS.

@media query position: fixed classList.toggle keydown Escape
styles.css + main.js 
/* CSS — Mobile nav */
@media (max-width: 768px) {
    .hamburger-btn { display: flex; }

    .main-nav {
        position: fixed;
        right: -100%; /* Oculto fuera de pantalla */
        width: 280px;
        height: 100vh;
        transition: right 0.3s ease;
    }
    .main-nav.nav-open { right: 0; }
}

/* JS — Funciones abrir/cerrar */
function openMenu() {
    mainNav.classList.add('nav-open');
    hamburgerBtn.classList.add('active');
    overlay.classList.add('active');
    document.body.style.overflow = 'hidden';
}
function closeMenu() {
    mainNav.classList.remove('nav-open');
    hamburgerBtn.classList.remove('active');
    overlay.classList.remove('active');
    document.body.style.overflow = '';
}
/* Cerrar con Escape */
document.addEventListener('keydown', e => {
    if (e.key === 'Escape' && isMenuOpen) closeMenu();
});

06
Carrusel de Productos
Navegación manual y auto-scroll — transform translateX
Explicación del algoritmo

El carrusel usa CSS transform: translateX() para desplazar las tarjetas sin reordenar el DOM. El contenedor padre tiene overflow: hidden para ocultar lo que está fuera de vista.

Algoritmo:
1. Se calcula el ancho de cada tarjeta + el gap entre ellas
2. Al avanzar: offset = -(index × (cardWidth + gap))
3. El índice cicla de 0 al total de slides con operador módulo %
4. Auto-play: setInterval avanza cada 5 segundos
5. Se pausa con mouseenter y se reanuda con mouseleave

El carrusel de selección premium usa animación CSS infinita (@keyframes) sin JS para máximo rendimiento.

transform translateX overflow hidden setInterval módulo %
main.js 
let currentIndex = 0;
const cardsPerView = 3;
const totalSlides = Math.ceil(cards.length / cardsPerView);

function updateSlider() {
    const cardWidth = cards[0].offsetWidth;
    const gap = 30;
    const offset = -(currentIndex * cardsPerView
                    * (cardWidth + gap));
    grid.style.transform =
        `translateX(${offset}px)`;
}

function nextSlide() {
    /* Módulo para ciclar infinitamente */
    currentIndex = (currentIndex + 1) % totalSlides;
    updateSlider();
}

/* Auto-play cada 5 segundos */
let autoplay = setInterval(nextSlide, 5000);

/* Pausar en hover */
slider.addEventListener('mouseenter', () =>
    clearInterval(autoplay));
slider.addEventListener('mouseleave', () =>
    autoplay = setInterval(nextSlide, 5000));

Formularios con Validación en Tiempo Real
Regex de email — Mensajes de estado — Accesibilidad
Explicación del algoritmo

Los formularios del sitio (newsletter, contacto, asesoría, registro) usan validación JavaScript antes de enviar, con retroalimentación visual inmediata.

Algoritmo de validación:
1. Se captura el evento click del botón (no el submit del form)
2. Se verifica que el campo no esté vacío (email.trim())
3. Se valida el formato con expresión regular: /^[^\s@]+@[^\s@]+\.[^\s@]+$/
4. Si falla: se añade clase .input-error y se muestra mensaje
5. Si pasa: se añade clase .input-success y se simula el envío
6. Los mensajes se eliminan automáticamente con setTimeout

RegExp classList :focus styles UX Feedback
main.js 
const emailRegex =
    /^[^\s@]+@[^\s@]+\.[^\s@]+$/;

submitBtn.addEventListener('click', (e) => {
    e.preventDefault();
    const email = emailInput.value.trim();

    /* Limpiar estados anteriores */
    emailInput.classList.remove(
        'input-error', 'input-success');

    /* Validar vacío */
    if (!email) {
        showError(emailInput,
            'Ingresa tu correo');
        return;
    }
    /* Validar formato */
    if (!emailRegex.test(email)) {
        showError(emailInput,
            'Correo no válido');
        return;
    }
    /* Éxito */
    showSuccess(emailInput, '¡Gracias!');
});

Sistema de Calificación — 5 Estrellas SVG
Rating interactivo con hover y selección
Explicación del algoritmo

Las estrellas de calificación están implementadas con SVG en color naranja corporativo en lugar de emojis, garantizando consistencia visual en todos los navegadores.

Algoritmo interactivo:
1. Cada estrella escucha eventos mouseover y click
2. Al pasar el cursor: se colorean todas las estrellas hasta la posición actual (efecto hover)
3. Al hacer clic: se fija la calificación en esa posición
4. Al salir el cursor sin seleccionar: vuelve a la calificación fijada
5. Se muestra un mensaje con la valoración elegida

En los testimonios, las estrellas tienen atributo role="img" y aria-label para accesibilidad.

mouseover click event SVG fill ARIA role="img"
JavaScript — Rating interactivo 
const estrellas = document
    .querySelectorAll('.estrella');
let calificacion = 0;

estrellas.forEach((star, index) => {

    star.addEventListener('mouseover', () => {
        colorear(index + 1);
    });

    star.addEventListener('mouseleave', () => {
        colorear(calificacion);
    });

    star.addEventListener('click', () => {
        calificacion = index + 1;
        mostrarMensaje(calificacion);
    });
});

function colorear(n) {
    estrellas.forEach((s, i) => {
        s.style.color = i < n
            ? '#E0994E'
            : '#ddd';
    });
}

09
Filtros de Categoría — Tienda
Filtrado dinámico por data attributes sin recargar página
Explicación del algoritmo

Los filtros de la tienda usan data attributes HTML5 para categorizar cada producto sin necesidad de base de datos ni recarga de página.

Algoritmo:
1. Cada tarjeta de producto tiene data-categoria="muscular" (o la categoría correspondiente)
2. Al hacer clic en un botón de filtro, se lee su data-categoria
3. Se recorre cada producto: si su categoría coincide → display: block; si no → display: none
4. La categoría "todos" siempre muestra todo
5. El contador de resultados se actualiza en tiempo real

data-categoria querySelectorAll dataset display none/block
tienda.html — data attribute + JS 
<!-- HTML: data-categoria en cada producto -->
<div class="tienda-producto-card"
     data-product-id="melena"
     data-categoria="cognitivo">

/* JS: algoritmo de filtrado */
filtros.forEach(function(btn) {
    btn.addEventListener('click',
    function() {
        var cat = this.dataset.categoria;
        var visibles = 0;

        productos.forEach(function(card) {
            if (cat === 'todos' ||
                card.dataset.categoria === cat) {
                card.style.display = '';
                visibles++;
            } else {
                card.style.display = 'none';
            }
        });
        contador.textContent =
            'Mostrando ' + visibles + ' resultados';
    });
});

10
Carrito de Compras con localStorage
Clase ShoppingCart — Persistencia de datos — Badge dinámico
Explicación del algoritmo

El carrito está implementado como una Clase JavaScript (ES6) llamada ShoppingCart. Usa localStorage para persistir los productos entre sesiones y páginas.

Métodos principales:
— loadCart(): Lee el carrito del localStorage al iniciar
— addItem(product): Agrega o incrementa cantidad si ya existe
— saveCart(): Serializa con JSON y guarda en localStorage
— updateCartBadge(): Actualiza el contador visual en el header
— renderCart(): Genera el HTML del carrito dinámicamente
— getTotal(): Calcula subtotal + IVA + envío

El badge del carrito se actualiza en todas las páginas porque la clase se instancia globalmente.

Class ES6 localStorage JSON.parse/stringify reduce()
main.js — Clase ShoppingCart 
class ShoppingCart {
    constructor() {
        this.items = this.loadCart();
        this.init();
    }

    loadCart() {
        const saved = localStorage
            .getItem('vitalityCart');
        return saved
            ? JSON.parse(saved) : [];
    }

    addItem(product) {
        const exists = this.items
            .find(i => i.id === product.id);
        if (exists) {
            exists.quantity++;
        } else {
            this.items.push(product);
        }
        this.saveCart();
    }

    saveCart() {
        localStorage.setItem(
            'vitalityCart',
            JSON.stringify(this.items));
        this.updateCartBadge();
    }

    getTotal() {
        return this.items.reduce(
            (t, i) => t + i.price * i.quantity, 0);
    }
}
const cart = new ShoppingCart();

SEO y Datos Estructurados — Schema.org
Meta tags · Open Graph · JSON-LD · Accesibilidad WCAG 2.1
Explicación

Cada página tiene un bloque completo de SEO codificado en el <head>. Incluye tres capas de optimización:

1. Meta tags básicos: title, description (150-160 chars), keywords, robots: index, follow y canonical para evitar contenido duplicado.

2. Open Graph: Permite que al compartir en redes sociales (Facebook, WhatsApp, LinkedIn) se muestre una vista previa con imagen, título y descripción personalizados.

3. Schema.org JSON-LD: Datos estructurados que los motores de búsqueda leen para entender el contenido. Cada página tiene su tipo específico: Organization (inicio), Product con AggregateRating (productos), ContactPage, Blog, etc.

meta tags Open Graph JSON-LD Schema.org WCAG 2.1
